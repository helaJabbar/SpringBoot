🧭 Plan global du webinar – Spring Boot REST API

👉 Objectif : construire une application complète de gestion (Users + Tasks) avec Spring Boot, JPA et PostgreSQL.

🟢 PARTIE 1 — Mise en place ⏱️ 20–30 min

✔ Objectif : Créer le projet et connecter la base de données

  • Création du projet Spring Boot (Spring Initializr)
  • Ajout des dépendances :
    • Spring Web
    • Spring Data JPA
    • PostgreSQL Driver
    • Lombok
  • Configuration du fichier application.properties

🟢 PARTIE 2 — Modélisation ⏱️ 30 min

✔ Objectif : Créer la structure des données (tables)

  • User Entity
  • Task Entity
  • Enum Status (ex: TODO, IN_PROGRESS, DONE)

🟢 PARTIE 3 — Repository Layer ⏱️ 10–15 min

✔ Objectif : Comprendre Spring Data JPA

  • TaskRepository
  • UserRepository
  • CRUD automatique (findAll, save, delete...)

🟢 PARTIE 4 — Service Layer (cœur métier) ⏱️ 30–40 min

✔ Objectif : Implémenter la logique métier

  • TaskService
  • UserService
  • Gestion des règles métier
  • Appels aux repositories

🟢 PARTIE 5 — REST API (Controller) ⏱️ 30–40 min

✔ Objectif : Exposer les endpoints REST

  • GET all
  • GET by id
  • POST create
  • PUT update
  • DELETE
  • Filtres : status, user, date

💡 Moment clé du webinar :

👉 Tester chaque endpoint en direct avec Postman

🟢 PARTIE 6 — Tests avec Postman ⏱️ 20 min

✔ Objectif : Valider toute l’application

  • Création utilisateur
  • Création tâche
  • Filtrage par status
  • Update status

🎯 Résultat final

✔ Une API REST complète
✔ Une architecture propre en couches
✔ Une base solide pour Spring Boot
