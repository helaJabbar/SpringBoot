Configuration d’une application Spring Boot

Structure d’un projet Spring Boot

Une application Spring Boot est organisée en plusieurs couches afin de séparer les responsabilités.

On retrouve généralement 4 packages principaux :

  • model
  • repository
  • service
  • controller

Rôle de chaque couche

📦 Model (Entity)

Contient les classes qui représentent les tables de la base de données. Chaque objet correspond à une ligne dans une table.

Le Model représente la structure des données de l’application. Il est directement lié à la base de données.

Chaque classe Model correspond à une table et chaque objet correspond à une ligne.

Rôle dans l’architecture :

  • Représenter les données métier
  • Stocker les informations de l’application
  • Être persisté en base de données

👉 Couche : Persistence / Data Layer

🗄 Repository

Permet d’accéder à la base de données. Il fournit les méthodes CRUD (save, findAll, findById, delete…).

Le Repository est responsable de la communication avec la base de données.

Il permet d’exécuter des opérations CRUD sans écrire de SQL grâce à Spring Data JPA.

Rôle dans l’architecture :

  • Accéder aux données
  • Interagir avec la base de données
  • Fournir des méthodes comme save(), findAll(), delete()

👉 Couche : Data Access Layer (DAL)

⚙ Service

Contient la logique métier de l’application. Il fait le lien entre Controller et Repository.

Le Service contient la logique métier de l’application.

C’est ici que les règles de gestion sont appliquées avant d’envoyer ou de récupérer les données.

Rôle dans l’architecture :

  • Appliquer les règles métier
  • Traiter les données
  • Faire le lien entre Controller et Repository

👉 Couche : Business Layer

🌐 Controller

Gère les requêtes HTTP (API REST). Il reçoit les appels du client et retourne les réponses.

Le Controller est le point d’entrée de l’application. Il reçoit les requêtes HTTP (GET, POST, PUT, DELETE).

Il ne contient pas de logique métier, il délègue tout au Service.

Rôle dans l’architecture :

  • Recevoir les requêtes HTTP
  • Envoyer les réponses au client
  • Appeler la couche Service

👉 Couche : Presentation Layer / API Layer

com.example.demo
│

spring.datasource.url=jdbc:postgresql://localhost:5432/ma_base
spring.datasource.username=postgres
spring.datasource.password=1234

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true

Résumé

Une bonne organisation en couches (Model, Repository, Service, Controller) permet de structurer proprement une application Spring Boot.
