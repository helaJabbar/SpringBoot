API REST avec Spring Boot

Une API REST permet à une application de communiquer avec d’autres systèmes (frontend, mobile, Postman…) en utilisant le protocole HTTP.

Spring Boot facilite la création d’API REST grâce à des annotations simples et une architecture en couches.

1. Qu’est-ce qu’une API REST ?

REST (Representational State Transfer) est un style d’architecture utilisé pour créer des services web. Les données sont échangées principalement en format JSON.

👉 Une API REST permet de manipuler des ressources (ex: utilisateurs, produits, étudiants…).

2. Les méthodes HTTP utilisées

  • GET → récupérer des données
  • POST → ajouter une nouvelle donnée
  • PUT → modifier une donnée complète
  • PATCH → modifier une donnée partiellement
  • DELETE → supprimer une donnée

3. Architecture d’une API REST

    Client (Browser / Postman / Frontend)
            ↓
    Controller (REST API)
            ↓
    Service (logique métier)
            ↓
    Repository (accès base de données)
            ↓
    Database

4. Création d’un Controller REST

En Spring Boot, une API REST est créée avec la classe Controller. 

    @RestController
    @RequestMapping("/api/users")
    public class UserController {
    
        @Autowired
        private UserService service;
    
        @GetMapping
        public List getAllUsers() {
            return service.findAll();
        }
    
        @GetMapping("/{id}")
        public User getUserById(@PathVariable Long id) {
            return service.findById(id);
        }
       
            @GetMapping("/search")
            public List searchByName(@RequestParam String name) {
                return service.findByName(name);
            }
            
        @PostMapping
        public User addUser(@RequestBody User user) {
            return service.save(user);
        }
    
        @DeleteMapping("/{id}")
        public void deleteUser(@PathVariable Long id) {
            service.delete(id);
        }
    }

5. Explication des annotations

  • @RestController → définit une API REST
  • @RequestMapping → définit le chemin principal
  • @GetMapping → récupération des données
  • @PostMapping → ajout de données
  • @DeleteMapping → suppression
  • @PathVariable → récupération d’un paramètre dans l’URL
  • @RequestBody → réception des données JSON
  • @RequestParam → permet de récupérer des données envoyées dans l’URL sous forme de paramètres.
  • @Autowired → injection du service

6. Exemple JSON envoyé (POST)

    {
        "name": "Ali",
        "email": "ali@gmail.com",
        "age": 22
    }

7. Test avec Postman

Postman est un outil qui permet de tester les API REST sans interface graphique.

  • GET → http://localhost:8080/api/users
  • GET by ID → http://localhost:8080/api/users/1
  • POST → envoi JSON
  • DELETE → suppression par ID

8. Résumé

Une API REST permet de créer une communication entre un client et un serveur. Spring Boot simplifie cela grâce aux annotations et à l’architecture en couches.
