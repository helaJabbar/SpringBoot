Serveur Tomcat dans Spring Boot

Spring Boot utilise un serveur intégré appelé Apache Tomcat. Cela permet d’exécuter l’application sans installer un serveur externe.

1. Qu’est-ce que Tomcat ?

Apache Tomcat est un serveur web Java qui permet d’exécuter des applications web Java. Il gère les requêtes HTTP et exécute les Servlets.

👉 Dans Spring Boot, Tomcat est embedded (intégré).

2. Rôle de Tomcat dans Spring Boot

  • Recevoir les requêtes HTTP
  • Gérer les connexions clients
  • Exécuter les controllers Spring
  • Retourner les réponses au navigateur ou Postman

3. Tomcat embarqué (Embedded Server)

Spring Boot intègre Tomcat automatiquement dans l’application. Cela signifie que tu n’as pas besoin de le configurer manuellement.

Avantage :

  • Déploiement simple
  • Aucune configuration externe
  • Application autonome (standalone)

4. Port par défaut

Par défaut, Tomcat utilise le port : 

8080

Exemple d’accès à une application : 

http://localhost:8080

5. Modifier le port Tomcat

Si le port 8080 est occupé, tu peux le changer dans : application.properties 

server.port=8081

L’application sera alors accessible sur : 

http://localhost:8081

6. Schéma simple du fonctionnement

Client (Browser / Postman)
        ↓
   Tomcat Server
        ↓
   Spring Controller
        ↓
   Service
        ↓
   Repository
        ↓
   Database

7. Résumé

Tomcat est le serveur intégré de Spring Boot qui permet de gérer les requêtes HTTP et d’exécuter l’application sans configuration externe.
