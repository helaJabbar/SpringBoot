JPA et Spring Data Repository

Rappel : Qu’est-ce que JPA ?

JPA (Java Persistence API) permet de manipuler une base de données en utilisant des objets Java. Chaque entité correspond à une table, et chaque objet correspond à une ligne.

1. Le Repository Spring Data JPA

Spring Data JPA simplifie l’accès aux données en fournissant automatiquement des méthodes prêtes à l’emploi. Il suffit d’étendre JpaRepository. 

        public interface UserRepository extends JpaRepository<User, Long> {
        }

👉 Sans écrire de SQL, Spring génère automatiquement les requêtes.

2. Les méthodes CRUD de base

Voici les méthodes principales déjà fournies par Spring :

  • save(entity) → ajouter ou modifier une entité
  • findAll() → récupérer tous les enregistrements
  • findById(id) → chercher par ID
  • deleteById(id) → supprimer par ID
  • delete(entity) → supprimer un objet

Exemple :

        userRepository.save(user);
        
        List<User> users = userRepository.findAll();
        
        Optional<User> u = userRepository.findById(1L);
        
        userRepository.deleteById(1L);

3. Les requêtes automatiques (Query Methods)

Spring Data permet de créer des requêtes automatiquement en utilisant le nom de la méthode.

Règles des Query Methods

Les méthodes comme findByName, findByEmail ou findByAge doivent respecter une règle de nommage précise.

Spring Data JPA analyse le nom de la méthode pour générer automatiquement la requête SQL correspondante.

Structure générale :

findBy + NomDuChamp + (Condition optionnelle)

Exemples valides :

  • findByName(String name) → recherche par nom
  • findByEmail(String email) → recherche par email
  • findByAge(int age) → recherche par âge
        public interface UserRepository extends JpaRepository<User, Long> {
        
            User findByEmail(String email);
        
            List<User> findByName(String name);
        }

4. Les requêtes avec conditions (Between, Like, etc.)

Spring Data JPA supporte des requêtes avancées basées sur le nom des méthodes.

Exemples :

  • findByAgeBetween(int min, int max)
  • findByNameContaining(String text)
  • findByEmailLike(String pattern)
        public interface UserRepository extends JpaRepository<User, Long> {
        
            List<User> findByAgeBetween(int min, int max);
        
            List<User> findByNameContaining(String text);
        
            List<User> findByEmailLike(String pattern);
        }

5. Exemple concret complet

        @GetMapping("/users")
        public List<User> getAllUsers() {
            return userRepository.findAll();
        }
        
        @GetMapping("/users/{id}")
        public User getUser(@PathVariable Long id) {
            return userRepository.findById(id).orElse(null);
        }
        
        @GetMapping("/users/search")
        public List<User> search(@RequestParam String name) {
            return userRepository.findByNameContaining(name);
        }
        
        @PostMapping("/users")
        public User addUser(@RequestBody User user) {
            return userRepository.save(user);
        }

6. Résumé

Spring Data JPA permet de manipuler une base de données sans écrire de SQL. Grâce aux méthodes comme save, findById, findBy, findBetween, le développement devient rapide et propre.
