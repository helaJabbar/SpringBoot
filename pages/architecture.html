L’architecture d’une application Spring Boot

Une architecture en couches

Une application Spring Boot peut être organisée de plusieurs façons. Le framework ne impose pas une structure unique, ce qui laisse une grande flexibilité aux développeurs.

Cependant, dans les applications web modernes, on utilise très souvent l’architecture n-tiers, car elle permet de séparer clairement les responsabilités du code.

Cette architecture peut contenir plusieurs couches, mais dans la majorité des cas, on retrouve une structure simple en 3 tiers :

  • Couche Présentation : interface utilisateur (Controller REST)
  • Couche Métier : logique applicative (Service)
  • Couche Accès aux données : interaction avec la base (Repository)
